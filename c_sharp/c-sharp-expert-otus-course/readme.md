### Значимые типы

1. Все примитивные типы (bool, int, double, ...)

2. record struct

3. struct

4. enum

**Особенности:**

1. Память для переменных этих типов выделяется в стэке

2. Являются потомками System.ValueType

3. Не наследуются. Не могут иметь virtual и abstruct методы. Все их методы являются неявно запечатанными

4. При присваивании копируются значения полей

5. Неупакованные значимые типы удаляются из стэка сразу, как будет возвращено управление предыдущему кадру стэка

6. Могут иметь методы, но ссылки на объект-тип у них нет, а следовательно и на таблицу методов, поэтому компилятор ищет объект-тип в неуправляемой памяти напрямую

7. Equals - сравнивает поля, GetHashCode учитывает значения полей

### Ссылочные типы

1. class

2. interface

3. delegate

4. object

5. record (record class)

**Особенности:**

1. Память для переменных этих типов выделяется в управляемой куче

2. Являются потомками System.Object

3. Наследуются

4. При присваивании копируется ссылка (адрес объекта в памяти)

5. Удаляются сборщиком мусора

5. Каждый объект, размещаемый в куче, содержит дополнительные члены, подлежащие инициализации

6. Могут иметь значение null

### Стэк

Кадр стэка содержит: параметры функции, локальные переменные, адрес возврата в вызвавшую функцию

**Особенности:**

1. Мгновенное выделение памяти - изменение значения регистра SPR (Stack Pointer Register)

2. Хорошо работает с кэшем процессора - является непрерывным блоком памяти, почти всегда находится в L1/L2

3. Прямая адресация - доступ к локальным переменным выполняется по фиксированному смещению относительно указателя кадра

**Вопросы:**
