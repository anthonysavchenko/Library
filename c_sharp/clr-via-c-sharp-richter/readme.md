## Алгоритм сборки мусора

1. При выполнении оператора new вычисляется размер объекта (все его поля, указатель на объект-тип и индекс блока синхронизации)

2. Если в поколении 0 управляемой кучи достаточно места для объекта, то переходим к шагу 2. Иначе переходим к сборке мусора на шаге 4

3. По адресу NextObjPtr обнуляется полученное количество байты, вызывается конструктор, указатель NextObjPtr сдвигается

4. Сборка мусора. Все потоки в процессе приостанавливаются. Переходим к этапу маркировки

5. Маркировка (marking phase). Всем объектам проставляется 0 в поле индекса блока синхронизации

6. Перебираются все корни и объекты, на которые они ссылаются. Им проставляется 1 в индекс блока синхронизации

7. Сжатие (compacting phase). Все "немусорные" объекты перемещаются влево (вниз), чтобы они занимали смежные блоки памяти. А ссылки на них и NextObjPtr корректируются соответственно

8. Если в результате сборки мусора не удалось освободить достаточно места для нового объекта, то получаем OutOfMemoryException

- Как алгоритм работает по поколениям? Во время маркировки перебираются объекты всех поколений?
